# Contributing Guide

Thank you for your interest in contributing to Discord AI Agent Ops Kit.

This project provides open-source templates, workflows, and examples for operating AI coding agents through Discord and other chat-based interfaces.

We welcome contributions that make AI-assisted development more practical, transparent, and useful for solo developers, small teams, educators, community builders, and open-source maintainers.

## Ways to contribute

You can contribute in several ways:

* Improve existing templates
* Add new workflow examples
* Fix unclear documentation
* Suggest better prompts for Codex or other AI coding agents
* Add examples for small OSS teams
* Add examples for non-engineers coordinating software projects
* Report issues or confusing parts of the repository

## Good first contributions

Good first contributions include:

* Improving wording in README.md
* Adding a new issue triage example
* Adding a new pull request review checklist item
* Creating a new Discord workflow example
* Improving the responsible AI agent use guide
* Adding practical examples for solo developers or educators

## How to contribute

1. Fork this repository.
2. Create a new branch for your change.
3. Make a focused update.
4. Open a pull request with a clear description.
5. Explain why the change is useful.

## Pull request guidelines

Please keep pull requests small and easy to review.

A good pull request should:

* Solve one clear problem
* Include a short explanation
* Avoid unnecessary complexity
* Update documentation when needed
* Be useful for maintainers, contributors, or small teams

## AI-generated contributions

AI tools such as Codex can be used to help draft documentation, examples, and workflow templates.

However, contributors are responsible for reviewing AI-generated content before submitting it.

Please check that AI-generated contributions are:

* Accurate
* Clear
* Safe to share publicly
* Free from sensitive information
* Useful for the project’s intended users

## Project values

This project values:

* Practicality
* Transparency
* Maintainer-friendly workflows
* Responsible AI use
* Clear documentation
* Small, reviewable contributions

## Questions and suggestions

If you are unsure whether your idea fits the project, please open an issue first and describe your suggestion.

We welcome practical ideas that help small teams and open-source maintainers use AI coding agents more effectively.
