# Weekly Maintainer Review Template

Use this template to review open-source maintenance work once a week.

## Purpose

This template helps maintainers organize issues, pull requests, documentation updates, and AI-assisted development tasks.

It is designed for solo developers, small teams, educators, and open-source maintainers who use Codex or other AI coding agents.

## Weekly review checklist

### 1. Review open issues

- Which issues need a response?
- Which issues can be closed?
- Which issues need more information?
- Which issues can Codex help summarize or prepare?

### 2. Review pull requests

- Which pull requests are ready for review?
- Which pull requests need changes?
- Are there any security or privacy concerns?
- Can Codex help prepare a review summary?

### 3. Review documentation

- Is the README still accurate?
- Are templates easy to understand?
- Are examples useful for new users?
- Are there missing guides or unclear sections?

### 4. Review release tasks

- Are there changes that should be included in the next release?
- Do release notes need to be drafted?
- Are there breaking changes?
- Are contributors credited properly?

### 5. Review AI-generated output

Before accepting AI-generated content, check:

- Is it accurate?
- Is it safe to publish?
- Is it clear enough for contributors?
- Does it expose sensitive information?
- Does it add unnecessary complexity?

## Suggested Codex prompt

Please help me review this repository's weekly maintenance status.

Focus on:

- open issues
- pull requests
- documentation gaps
- release note preparation
- possible security or privacy concerns
- small improvements that would help contributors

Return a concise maintainer action plan.
