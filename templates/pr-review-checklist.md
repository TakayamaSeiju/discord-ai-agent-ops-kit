# Pull Request Review Checklist

Use this checklist before asking Codex or a human maintainer to review a pull request.

## Purpose

This template helps maintainers review pull requests more consistently and efficiently.

## Checklist

- Does the pull request solve a clearly described issue?
- Are the changed files appropriate for the task?
- Is the implementation easy to understand?
- Are tests, examples, or documentation included where needed?
- Is the change small enough to review safely?
- Are there any security or privacy risks?
- Does the pull request introduce unnecessary complexity?
- Can the maintainer verify the result quickly?

## Suggested Codex review prompt

Please review this pull request for correctness, maintainability, documentation quality, and possible security or privacy issues.

Focus on practical risks and suggest a concise improvement plan.

## Review output format

Please return:

1. Summary
2. Strengths
3. Possible issues
4. Suggested improvements
5. Final recommendation
