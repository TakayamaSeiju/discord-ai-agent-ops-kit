# Release Note Template

Use this template to create clear release notes for small open-source projects.

## Version

v0.0.0

## Release date

YYYY-MM-DD

## Summary

Write a short summary of this release in one or two sentences.

## Added

- New features or new templates

## Changed

- Improvements to existing workflows, documentation, or examples

## Fixed

- Bug fixes or corrections

## Documentation

- README updates
- New guides
- Improved examples

## Breaking changes

- List any breaking changes, or write "None"

## Maintainer notes

Explain anything contributors or users should know before using this release.

## Suggested Codex prompt

Please draft release notes for this repository based on the recent changes.

Focus on:

- clear summary
- user-facing changes
- documentation updates
- possible breaking changes
- concise maintainer notes
