# Getting Started

This guide explains how to start using Discord AI Agent Ops Kit.

## 1. Choose your use case

First, decide how you want to use AI coding agents.

Common use cases include:

- assigning coding tasks to Codex
- organizing documentation updates
- triaging GitHub issues
- preparing pull request reviews
- drafting release notes
- coordinating small team development through Discord

## 2. Set up your Discord workspace

Create a few channels for AI-assisted development.

Suggested channels:

- #ideas
- #coding-tasks
- #issue-triage
- #codex-review
- #documentation
- #release-notes

Each channel should have a clear purpose so that tasks and reviews are easy to track.

## 3. Use the templates

Start with the templates in the `templates/` folder.

Recommended first templates:

- `codex-task-request.md`
- `issue-triage-prompt.md`
- `pr-review-checklist.md`
- `release-note-template.md`

Copy the template, fill in the details, and use it when asking Codex or another AI coding agent for help.

## 4. Review AI-generated output

AI-generated code and documentation should always be reviewed by a human maintainer.

Before accepting output, check:

- Is it accurate?
- Is it easy to understand?
- Is it safe to publish?
- Does it expose sensitive information?
- Does it add unnecessary complexity?
- Is it useful for future contributors?

## 5. Keep changes small

Small changes are easier to review.

When using Codex or another AI coding agent, ask for focused outputs such as:

- one documentation update
- one issue summary
- one checklist improvement
- one release note draft
- one small code change

## 6. Improve the workflow over time

This project is designed to grow through practical use.

As you use these templates, you can improve them by:

- adding examples
- clarifying instructions
- improving review checklists
- creating new workflows
- documenting mistakes and lessons learned
